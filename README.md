# 📡 SV Fault Injector & Publisher (IEC 61850-9-2)

Утилита для генерации и публикации потоков Sampled Values (SV) в соответствии со стандартом IEC 61850-9-2. Разработана на базе `libiec61850 v1.6.1` и предназначена для тестирования защитных реле, PMU, регистраторов и сетевой инфраструктуры энергетических объектов. Поддерживает инжекцию сетевых аномалий (потеря и перестановка пакетов) с сохранением точной временной синхронизации.

---

## 1. Описание программы
Программа эмулирует работу объединяющего устройства (Merging Unit) и публикует SV-кадры на канальном уровне (Ethernet II / 802.1Q). Конфигурация потока строго соответствует заданному CID-файлу: MAC-адрес назначения, VLAN, APPID, `smvID`, `confRev`, структура DataSet (3 фазы тока `INT32` + `Quality`). Генератор использует zero-copy запись данных в предварительно выделенный буфер, что гарантирует минимальные задержки и стабильный джиттер.

---

## 2. Особенности
✅ **Точное соответствие CID** – все сетевые и прикладные параметры читаются из конфигурации устройства.  
✅ **Высокоточный тайминг** – используется `clock_nanosleep()` с абсолютным временем (`CLOCK_MONOTONIC`), предотвращающий накопление погрешности.  
✅ **Real-time оптимизации** – автоматическое переключение на планировщик `SCHED_FIFO` (приоритет 50) и блокировка памяти `mlockall()` для предотвращения page faults.  
✅ **Инжекция искажений** – встроенные режимы `DROP` (эмуляция потери пакетов) и `REORDER` (эмуляция задержки/перестановки с сохранением оригинальной метки времени `RefrTm`).  
✅ **Диагностика** – пошаговый вывод инициализации с `fflush(stdout)` для мгновенного выявления причин падения.  
✅ **Zero-copy архитектура** – данные записываются напрямую в Ethernet-буфер без динамических выделений памяти в цикле публикации.

| Компонент | Версия / Примечание |
|-----------|---------------------|

| Компилятор | GCC 10+ или Clang (стандарт C11) |
| Зависимости | `libiec61850-dev` / `libiec61850` (v1.6.1), `libpthread`, `librt`, `libm` |

## 4. Сборка
Стандартный workflow через CMake:
```bash
# 1. Создание директории сборки
mkdir -p build && cd build

# 2. Конфигурация (автоматически находит заголовки и библиотеку)
cmake .. -DCMAKE_BUILD_TYPE=Release

# 3. Компиляция
make -j$(nproc)
```
📌 **Примечание:** Если `libiec61850` установлена в нестандартный путь, укажите его явно:
```bash
cmake .. -DCMAKE_PREFIX_PATH=/opt/libiec61850 -DLIBIEC61850_INC_DIR=/opt/libiec61850/include
```

## 5. Использование
Базовый запуск требует прав суперпользователя для доступа к канальному уровню:
```bash
sudo ./sv_generator <interface>
```
Пример:
```bash
sudo ./sv_generator enp0s3
```
Программа выведет этапы инициализации и перейдёт в цикл публикации. Остановка выполняется стандартным сочетанием `Ctrl+C` (обработка `SIGINT` с корректным освобождением ресурсов).

🔍 **Проверка в Wireshark:**
- Фильтр: `sv` или `eth.type == 0x88ba`
- Рекомендуемые колонки: `sv.smpCnt`, `sv.smpRate`, `sv.confRev`, `Time delta from previous displayed packet`
- Ожидаемый интервал между кадрами: `~1.667 мс` (для `smpRate=12`, 50 Гц)

## 6. Параметры командной строки
| Параметр | Синтаксис | Описание |
|----------|-----------|----------|
| `--iface` / `-i` | `-i <iface>` | Имя сетевого интерфейса (альтернатива позиционному аргументу) |
| `--drop` | `--drop <START> <COUNT>` | Пропустить `COUNT` пакетов, начиная с `SmpCnt=START`. Имитация потери кадров в сети. |
| `--reorder` | `--reorder <DELAY> <BEFORE>` | Задержать пакет `SmpCnt=DELAY` и вставить его перед `SmpCnt=BEFORE`. Сохраняет оригинальную метку времени. |
| `--help` / `-h` | `-h` | Вывод справки и завершение работы |

### Примеры:
```bash
# Чистый поток на интерфейсе eth1
sudo ./sv_generator eth1

# Пропустить 3 пакета, начиная с SmpCnt=10
sudo ./sv_generator enp0s3 --drop 10 3

# Задержать пакет #5, вставить перед #15
sudo ./sv_generator enp0s3 --reorder 5 15

# Комбинированная инъекция
sudo ./sv_generator eth0 --drop 20 2 --reorder 7 14
```

## 7. Инструкция по корректировке при изменении SmpRate
Значение `smpRate` задаётся на этапе компиляции для обеспечения детерминированного поведения и отсутствия накладных расходов в реальном времени. При изменении `smpRate` в CID-файле необходимо отредактировать **5 мест** в `main.c` и пересобрать проект.

Формула расчёта (для частоты сети 50 Гц):
`samples_per_sec = smpRate × 50`

| № | Место в коде | Что изменить | Формула / Пример (smpRate=80) |
|---|--------------|--------------|-------------------------------|
| 1 | `SVPublisher_ASDU_setSmpRate(asdu, ...)` | Указать новое значение | `SVPublisher_ASDU_setSmpRate(asdu, 80);` |
| 2 | `SVPublisher_ASDU_setSmpCntWrap(asdu, ...)` | Предел обёртки счётчика | `SVPublisher_ASDU_setSmpCntWrap(asdu, 4000);` *(80×50)* |
| 3 | `const long INTERVAL_NS = ...` | Интервал между кадрами (нс) | `const long INTERVAL_NS = 250000;` *(1e9 / 4000)* |
| 4 | `sample_idx = (sample_idx + 1) % ...` | Модуль инкремента | `sample_idx = (sample_idx + 1) % 4000;` |
| 5 | `double angle = 2.0 * M_PI * ((double)(curr % ...) / ...)` | Делитель фазы синусоиды | `double angle = 2.0 * M_PI * ((double)(curr % 80) / 80.0);` |

 **Алгоритм быстрого пересчёта:**
1. Определите новый `smpRate` из CID.
2. Вычислите `WRAP = smpRate * 50`.
3. Вычислите `INTERVAL_NS = 1000000000 / WRAP`.
4. Замените значения в строках 1–5 таблицы выше.
5. Выполните `make -j$(nproc)` и запустите утилиту.

> ⚠️ **Важно:** Для частоты сети 60 Гц формула меняется на `WRAP = smpRate * 60`. Остальные шаги идентичны.
